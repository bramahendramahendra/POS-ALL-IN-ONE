# Prompt 3.24 — Offline Mode: SQLite Lokal

## File Baru / Diubah
`desktop/src/js/db-local.js`, `desktop/main.js`, `desktop/preload.js`

## Konsep
Saat offline, operasi kasir (transaksi & kas harian) diarahkan ke SQLite lokal via Electron IPC.
Modul lain (produk, pelanggan, dll.) tetap bisa dibaca dari cache lokal, tapi tidak bisa diubah saat offline.

## Skema SQLite Lokal (tabel yang tetap dipertahankan)

```sql
-- Tabel yang dibutuhkan untuk operasi offline
CREATE TABLE IF NOT EXISTS local_transactions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    local_id    TEXT UNIQUE NOT NULL,    -- UUID lokal, belum ada server ID
    data        TEXT NOT NULL,           -- JSON payload transaksi
    created_at  TEXT NOT NULL,
    sync_status TEXT DEFAULT 'PENDING'   -- PENDING | SYNCED
);

CREATE TABLE IF NOT EXISTS local_cash_drawer (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    date        TEXT UNIQUE NOT NULL,
    opened_at   TEXT,
    closed_at   TEXT,
    opening_balance REAL DEFAULT 0,
    data        TEXT NOT NULL            -- JSON snapshot
);

-- Cache produk untuk kasir offline
CREATE TABLE IF NOT EXISTS cache_products (
    id          INTEGER PRIMARY KEY,
    data        TEXT NOT NULL,           -- JSON produk
    updated_at  TEXT NOT NULL
);
```

## IPC Handlers di main.js

```javascript
// Simpan transaksi offline
ipcMain.handle('local:saveTransaction', async (event, transactionData) => {
    const localId = require('crypto').randomUUID();
    db.prepare(`INSERT INTO local_transactions (local_id, data, created_at) VALUES (?, ?, ?)`)
      .run(localId, JSON.stringify(transactionData), new Date().toISOString());
    return { local_id: localId };
});

// Ambil transaksi offline pending
ipcMain.handle('local:getPendingTransactions', async () => {
    return db.prepare(`SELECT * FROM local_transactions WHERE sync_status = 'PENDING' ORDER BY created_at`).all();
});

// Update cache produk
ipcMain.handle('local:cacheProducts', async (event, products) => {
    const insert = db.prepare(`INSERT OR REPLACE INTO cache_products (id, data, updated_at) VALUES (?, ?, ?)`);
    const insertMany = db.transaction((items) => {
        for (const p of items) insert.run(p.id, JSON.stringify(p), new Date().toISOString());
    });
    insertMany(products);
    return true;
});

// Baca cache produk
ipcMain.handle('local:getCachedProducts', async () => {
    return db.prepare(`SELECT data FROM cache_products`).all().map(r => JSON.parse(r.data));
});
```

## Preload.js — Expose ke Renderer

```javascript
contextBridge.exposeInMainWorld('localDB', {
    saveTransaction:       (data)     => ipcRenderer.invoke('local:saveTransaction', data),
    getPendingTransactions: ()        => ipcRenderer.invoke('local:getPendingTransactions'),
    cacheProducts:         (products) => ipcRenderer.invoke('local:cacheProducts', products),
    getCachedProducts:     ()         => ipcRenderer.invoke('local:getCachedProducts'),
});
```

## Integrasi di Kasir (kasir.js)

```javascript
async function createTransaction(data) {
    if (!connectionMonitor.isOnline) {
        // Simpan ke SQLite lokal
        const result = await window.localDB.saveTransaction(data);
        showNotification('Transaksi disimpan offline. Akan disinkronkan saat online.', 'warning');
        return { ...data, id: result.local_id, is_offline: true };
    }
    // Online — kirim ke backend seperti biasa
    return await apiClient.post('/transactions', data);
}
```

## Output yang Diharapkan
- Kasir tetap bisa membuat transaksi saat offline (tersimpan ke SQLite)
- Produk bisa dibaca dari cache lokal saat offline
- Data offline siap diambil oleh Sync Engine (3.27)
