# Prompt 3.27 — Sync Engine: Proses Antrian

## File Baru
`desktop/src/js/sync-engine.js`

## Konsep
Sync Engine memproses item `PENDING` satu per satu secara berurutan (FIFO). 
Dijalankan otomatis saat koneksi pulih (`connection:restored`) dan bisa dipicu manual dari Sync Center.

## Implementasi

```javascript
// sync-engine.js
class SyncEngine {
    constructor() {
        this.isRunning = false;
    }

    // Endpoint map per entity + action
    _getEndpoint(entity, action, item) {
        const map = {
            'transaction:create':   { method: 'POST', url: '/transactions' },
            'expense:create':       { method: 'POST', url: '/expenses' },
            'cash_drawer:open':     { method: 'POST', url: '/cash-drawer/open' },
            'cash_drawer:close':    { method: 'POST', url: `/cash-drawer/${item.server_id}/close` },
        };
        return map[`${entity}:${action}`] || null;
    }

    async start() {
        if (this.isRunning || !connectionMonitor.isOnline) return;
        this.isRunning = true;

        try {
            const pendingItems = await window.syncQueue.getPending();
            for (const item of pendingItems) {
                if (!connectionMonitor.isOnline) break; // berhenti jika offline lagi

                await this._processItem(item);
            }
        } finally {
            this.isRunning = false;
            window.dispatchEvent(new CustomEvent('sync:completed'));
        }
    }

    async _processItem(item) {
        // Tandai sebagai SYNCING
        await window.syncQueue.updateStatus({ id: item.id, status: 'SYNCING' });

        const endpoint = this._getEndpoint(item.entity, item.action, item);
        if (!endpoint) {
            await window.syncQueue.updateStatus({ id: item.id, status: 'FAILED', errorMsg: 'Unknown entity/action' });
            return;
        }

        try {
            const payload = JSON.parse(item.payload);
            const response = await apiClient[endpoint.method.toLowerCase()](endpoint.url, payload);

            // Update sync_status di local_transactions jika ada
            if (item.entity === 'transaction' && item.local_id) {
                await window.localDB.markTransactionSynced(item.local_id, response.id);
            }

            // Tandai SYNCED
            await window.syncQueue.updateStatus({ id: item.id, status: 'SYNCED' });

        } catch (error) {
            const retryCount = (item.retry_count || 0) + 1;
            const newStatus = retryCount >= 3 ? 'FAILED' : 'PENDING';
            await window.syncQueue.updateStatus({
                id: item.id,
                status: newStatus,
                errorMsg: error.message || 'Sync failed',
            });
            // Jika FAILED setelah 3x retry, lanjut ke item berikutnya
        }
    }
}

const syncEngine = new SyncEngine();

// Jalankan otomatis saat koneksi pulih
window.addEventListener('connection:restored', () => {
    setTimeout(() => syncEngine.start(), 1000); // delay 1 detik agar koneksi stabil
});
```

## Tambahan di main.js — Mark Synced

```javascript
ipcMain.handle('local:markTransactionSynced', async (event, { localId, serverId }) => {
    db.prepare(`UPDATE local_transactions SET sync_status = 'SYNCED' WHERE local_id = ?`).run(localId);
    db.prepare(`UPDATE sync_queue SET server_id = ? WHERE local_id = ?`).run(serverId, localId);
    return true;
});
```

## Urutan Prioritas Sync
1. `cash_drawer:open` — kas harus terbuka sebelum transaksi diproses
2. `transaction:create` — transaksi kasir
3. `expense:create` — pengeluaran
4. `cash_drawer:close` — tutup kas terakhir

## Output yang Diharapkan
- Antrian PENDING diproses otomatis saat online
- Setiap item diproses satu per satu — item gagal tidak memblokir item berikutnya
- Status diperbarui realtime (bisa dilihat di Sync Center)
