# Prompt 3.28 — Sync Engine: Resume Jika Terputus

## File Diubah
`desktop/src/js/sync-engine.js`, `desktop/main.js`

## Masalah yang Diselesaikan
Jika koneksi terputus saat item sedang diproses (status = `SYNCING`), item tersebut tertinggal di status SYNCING selamanya — tidak dilanjutkan, tidak di-retry.

## Solusi
Saat aplikasi **pertama kali load** atau **koneksi pulih kembali**, cek apakah ada item dengan status `SYNCING`. Jika ada, rollback item tersebut ke `PENDING` dan lanjutkan dari item itu.

## Implementasi

### Tambahan di main.js — Reset SYNCING saat startup

```javascript
// Jalankan sekali saat aplikasi mulai
function resetStuckSyncItems() {
    const count = db.prepare(`
        UPDATE sync_queue SET status = 'PENDING', updated_at = ?
        WHERE status = 'SYNCING'
    `).run(new Date().toISOString()).changes;

    if (count > 0) {
        console.log(`[SyncEngine] Reset ${count} item SYNCING → PENDING (interrupted)`);
    }
}

// Panggil saat app.whenReady
app.whenReady().then(() => {
    resetStuckSyncItems();
    createWindow();
});
```

### Tambahan di sync-engine.js — Reset sebelum mulai sync

```javascript
class SyncEngine {
    async start() {
        if (this.isRunning || !connectionMonitor.isOnline) return;

        // Reset item yang tertinggal di SYNCING (dari sesi sebelumnya yang terputus)
        await window.syncQueue.resetSyncing();

        this.isRunning = true;
        // ... lanjut proses seperti 3.27
    }
}
```

### IPC Handler di main.js

```javascript
// Reset semua item SYNCING → PENDING
ipcMain.handle('syncQueue:resetSyncing', async () => {
    const result = db.prepare(`
        UPDATE sync_queue SET status = 'PENDING', updated_at = ?
        WHERE status = 'SYNCING'
    `).run(new Date().toISOString());
    return result.changes; // jumlah item yang di-reset
});
```

### Update Preload.js

```javascript
contextBridge.exposeInMainWorld('syncQueue', {
    // ... handler sebelumnya dari 3.25
    resetSyncing: () => ipcRenderer.invoke('syncQueue:resetSyncing'),
});
```

## Skenario Resume

```
Antrian:
1. expense:create    → SYNCED ✓
2. transaction:create → SYNCED ✓
3. transaction:create → SYNCING ← koneksi terputus di sini
4. transaction:create → PENDING
5. expense:create    → PENDING

Saat startup / koneksi pulih:
→ Item 3 direset: SYNCING → PENDING
→ Sync Engine mulai dari item 3
→ Item 3, 4, 5 diproses berurutan
```

## Output yang Diharapkan
- Tidak ada item yang "stuck" di status SYNCING selamanya
- Item yang terputus dilanjutkan dari titik terputus (bukan dari awal)
- Item yang sudah SYNCED tidak diproses ulang
