# Prompt 3.26 — Antrian Sync: Simpan ke Antrian

## File Diubah
`desktop/src/js/kasir.js`, `desktop/src/js/finance.js` (pengeluaran), `desktop/src/js/kas-harian.js`

## Konsep
Setiap operasi yang dilakukan saat **offline** langsung dimasukkan ke `sync_queue` dengan status `PENDING`.

## Implementasi per Modul

### Kasir — Buat Transaksi Offline

```javascript
// kasir.js
async function createTransaction(payload) {
    if (!connectionMonitor.isOnline) {
        const localId = crypto.randomUUID();
        const transactionData = { ...payload, device_source: 'desktop', local_id: localId };

        // Simpan ke local_transactions
        await window.localDB.saveTransaction(transactionData);

        // Tambah ke sync_queue
        await window.syncQueue.add({
            entity:  'transaction',
            action:  'create',
            localId: localId,
            payload: transactionData,
        });

        showNotification('Transaksi disimpan offline', 'warning');
        return { success: true, local_id: localId, is_offline: true };
    }
    return await apiClient.post('/transactions', payload);
}
```

### Pengeluaran — Buat Offline

```javascript
// finance.js (bagian pengeluaran)
async function createExpense(payload) {
    if (!connectionMonitor.isOnline) {
        const localId = crypto.randomUUID();
        await window.syncQueue.add({
            entity:  'expense',
            action:  'create',
            localId: localId,
            payload: { ...payload, local_id: localId },
        });
        showNotification('Pengeluaran disimpan offline', 'warning');
        return { success: true, local_id: localId, is_offline: true };
    }
    return await apiClient.post('/expenses', payload);
}
```

### Kas Harian — Buka/Tutup Offline

```javascript
// kas-harian.js
async function openCashDrawer(payload) {
    if (!connectionMonitor.isOnline) {
        await window.syncQueue.add({
            entity:  'cash_drawer',
            action:  'open',
            payload: { ...payload, date: new Date().toISOString().slice(0, 10) },
        });
        showNotification('Kas harian dibuka offline', 'warning');
        return { success: true, is_offline: true };
    }
    return await apiClient.post('/cash-drawer/open', payload);
}
```

## Catatan Penting
- Hanya operasi **create** yang disimpan ke antrian offline
- Operasi `update` dan `delete` di entitas master (produk, pelanggan, dll.) **tidak diizinkan saat offline** — tampilkan pesan error
- `local_id` menggunakan `crypto.randomUUID()` — tersedia di Electron/modern browser
- Setiap item antrian bersifat **idempoten** — jika dikirim ulang tidak boleh duplikat di server

## Output yang Diharapkan
Semua operasi offline (transaksi, pengeluaran, kas harian) tersimpan di `sync_queue` dengan status PENDING dan siap diproses oleh Sync Engine (3.27).
