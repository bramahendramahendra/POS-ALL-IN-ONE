# Prompt 4.14 — Deploy ke VPS

## File Baru
`web/nginx.conf`, `backend/deploy/docker-compose.yml`, `backend/deploy/Dockerfile`, `.github/workflows/deploy-backend.yml`

## Struktur Deployment

```
VPS
├── /var/www/pos-web/     ← Static files web app (HTML/CSS/JS)
├── /opt/pos-backend/     ← Binary Go backend
└── MySQL 8.0             ← Database
```

## 1. Dockerfile — Backend Go

```dockerfile
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN go build -o pos-backend ./cmd/main.go

FROM alpine:latest
RUN apk --no-cache add ca-certificates tzdata
WORKDIR /root/
COPY --from=builder /app/pos-backend .
COPY --from=builder /app/config.json .
EXPOSE 8080
CMD ["./pos-backend"]
```

## 2. docker-compose.yml

```yaml
version: '3.8'
services:
  backend:
    build: .
    restart: always
    ports:
      - "8080:8080"
    environment:
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_USER=${DB_USER}
      - DB_PASS=${DB_PASS}
      - DB_NAME=${DB_NAME}
      - JWT_SECRET=${JWT_SECRET}
    depends_on:
      - mysql

  mysql:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASS}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASS}
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:
```

## 3. nginx.conf

```nginx
server {
    listen 80;
    server_name namadomain.com www.namadomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name namadomain.com www.namadomain.com;

    ssl_certificate     /etc/letsencrypt/live/namadomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/namadomain.com/privkey.pem;

    # Web App (static files)
    root /var/www/pos-web;
    index index.html;
    try_files $uri $uri/ /index.html;

    # Backend API
    location /api/ {
        proxy_pass         http://localhost:8080/;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}
```

## 4. GitHub Actions — Auto Deploy Backend

```yaml
# .github/workflows/deploy-backend.yml
name: Deploy Backend

on:
  push:
    branches: [main]
    paths: ['backend/**']

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host:     ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key:      ${{ secrets.VPS_SSH_KEY }}
          script: |
            cd /opt/pos-backend
            git pull origin main
            docker compose up -d --build backend
```

## Checklist Deploy Manual

```bash
# 1. Install dependencies di VPS
apt update && apt install -y nginx certbot python3-certbot-nginx docker.io docker-compose

# 2. Setup SSL
certbot --nginx -d namadomain.com

# 3. Copy web files
rsync -avz web/ user@vps:/var/www/pos-web/

# 4. Jalankan backend
cd /opt/pos-backend && docker compose up -d
```

## Catatan
- Ganti semua `namadomain.com` dengan domain aktual
- Simpan kredensial di `.env` file di VPS, jangan di repo
- Backup MySQL rutin via cronjob

## Output yang Diharapkan
Web app dan backend ter-deploy di VPS, dapat diakses via HTTPS dari browser manapun.
