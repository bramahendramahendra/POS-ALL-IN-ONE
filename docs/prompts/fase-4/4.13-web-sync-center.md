# Prompt 4.13 — Web Halaman Sync Center

## File
`web/sync-center.html`, `web/src/js/sync-center.js`

## Guard Auth
```javascript
requireAuth();
requireRole('owner', 'admin');
```

## Perbedaan dengan Desktop Sync Center (3.22)

| Fitur | Desktop | Web |
|---|---|---|
| Tab Konflik Transaksi | Ada + Approve/Reject | Ada + Approve/Reject |
| Tab Antrian Sync | Ada (local queue) | **Tidak ada** (web tidak punya antrian lokal) |
| Tab Riwayat Sync | Ada | Ada (read-only) |
| Indikator koneksi | Ada | Tidak ada (web selalu online) |

## Implementasi

### Tab 1 — Konflik Transaksi (identik dengan desktop)

```javascript
// Ambil konflik
const conflicts = await apiClient.get('/sync/conflicts', { status: 'pending', page, limit });

// Resolve konflik
await apiClient.post(`/sync/conflicts/${conflictId}/resolve`, { action: 'approve' });
await apiClient.post(`/sync/conflicts/${conflictId}/resolve`, { action: 'reject' });
```

### Tab 2 — Riwayat Sync

```javascript
const history = await apiClient.get('/sync/history', { start_date, end_date, device_source, page, limit });
// Tampilkan: tanggal, device (desktop/web/android), jumlah item, status, durasi
```

## Catatan
- Web hanya bisa **melihat dan resolve** konflik, tidak membuat konflik
- Konflik hanya muncul dari device desktop yang sync data offline
- Owner/Admin di web bisa approve/reject konflik dari device desktop manapun

## Output yang Diharapkan
Sync Center web berfungsi untuk melihat dan resolve konflik dari perangkat desktop.
