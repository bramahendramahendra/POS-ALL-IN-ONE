# Prompt 2.3 — Middleware Single Active Session & Role-Based Access

## Konteks
Dua middleware penting yang harus dibuat sebelum domain lainnya:
1. **BearerAuth** — verifikasi token + cek single active session
2. **RoleAccess** — batasi akses endpoint berdasarkan role

Middleware yang sudah ada (`middleware/auth/bearer_auth_middleware.go`) perlu disesuaikan untuk POS.

## 1. Update Bearer Auth Middleware

File: `middleware/auth/bearer_auth_middleware.go`

```go
func BearerAuthMiddleware(authService auth.AuthService) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. Ambil token dari header: Authorization: Bearer <token>
        authHeader := c.GetHeader("Authorization")
        if authHeader == "" || !strings.HasPrefix(authHeader, "Bearer ") {
            c.Error(&errors.UnauthenticatedError{Message: "Token tidak ditemukan"})
            c.Abort()
            return
        }

        token := strings.TrimPrefix(authHeader, "Bearer ")

        // 2. Validasi token: verify JWT + cek di tabel sessions
        session, err := authService.ValidateToken(token)
        if err != nil {
            // Bedakan pesan error:
            // - Token expired biasa
            // - Token tidak ada di sessions (sudah login di device lain)
            c.Error(&errors.UnauthenticatedError{Message: err.Error()})
            c.Abort()
            return
        }

        // 3. Set user info ke context untuk dipakai handler
        c.Set("user_id", session.UserID)
        c.Set("user_role", session.UserRole)   // tambahkan role ke session
        c.Set("device_info", session.DeviceInfo)
        c.Set("token", token)

        c.Next()
    }
}
```

## 2. Buat Role Access Middleware

File: `middleware/role_middleware.go`

```go
// RoleMiddleware membatasi akses berdasarkan role
// Contoh penggunaan: RoleMiddleware("owner", "admin")
func RoleMiddleware(allowedRoles ...string) gin.HandlerFunc {
    return func(c *gin.Context) {
        role, exists := c.Get("user_role")
        if !exists {
            c.Error(&errors.UnauthenticatedError{Message: "Unauthorized"})
            c.Abort()
            return
        }

        userRole := role.(string)
        for _, allowed := range allowedRoles {
            if userRole == allowed {
                c.Next()
                return
            }
        }

        c.Error(&errors.UnauthorizededError{Message: "Anda tidak memiliki akses ke fitur ini"})
        c.Abort()
    }
}
```

## 3. Helper: Get User dari Context

File: `helper/context_helper.go`

```go
func GetUserID(c *gin.Context) int {
    userID, _ := c.Get("user_id")
    return userID.(int)
}

func GetUserRole(c *gin.Context) string {
    role, _ := c.Get("user_role")
    return role.(string)
}

func GetDeviceInfo(c *gin.Context) string {
    device, _ := c.Get("device_info")
    return device.(string)
}
```

## 4. Contoh Penggunaan di Routes

```go
// Protected routes (semua role)
protected := r.Group("/api")
protected.Use(auth_middleware.BearerAuthMiddleware(authService))
{
    protected.GET("/products", productHandler.GetAll)
    protected.POST("/transactions", transactionHandler.Create)

    // Hanya owner & admin
    ownerAdmin := protected.Group("/")
    ownerAdmin.Use(middleware.RoleMiddleware("owner", "admin"))
    {
        ownerAdmin.POST("/products", productHandler.Create)
        ownerAdmin.PUT("/products/:id", productHandler.Update)
        ownerAdmin.DELETE("/products/:id", productHandler.Delete)
        ownerAdmin.GET("/sync/conflicts", syncHandler.GetConflicts)
    }

    // Hanya admin
    adminOnly := protected.Group("/")
    adminOnly.Use(middleware.RoleMiddleware("admin"))
    {
        adminOnly.POST("/settings/reset", settingHandler.Reset)
        adminOnly.POST("/restore", backupHandler.Restore)
    }
}
```

## 5. Update Session Model
Pastikan tabel `sessions` dan model menyimpan `role` user:
```sql
ALTER TABLE sessions ADD COLUMN user_role VARCHAR(20) NOT NULL DEFAULT 'kasir';
```

## Output yang Diharapkan
- Semua endpoint protected memerlukan JWT yang valid
- Token tidak ada di sessions → 401 "Sesi berakhir karena login di perangkat lain"
- Role tidak sesuai → 403 "Tidak memiliki akses"
- `user_id`, `user_role`, `device_info` tersedia di semua handler via context
