# Prompt 2.1 — Migration Database MySQL

## Konteks
Backend Go GIN sudah ada dengan struktur domain-driven design.
Database menggunakan GORM + MySQL dengan raw SQL queries.
Skema lengkap sudah dirancang di prompt 1.1.

## Tugas
Buat sistem migration database untuk project POS.

### 1. Buat folder `backend/database/`
```
backend/database/
├── migrations/
│   ├── 001_init_schema.sql    ← DDL semua tabel (dari prompt 1.1)
│   └── 002_seed_data.sql      ← Data awal
└── migrate.go                 ← Script runner migration
```

### 2. Buat `backend/database/migrate.go`
```go
package database

import (
    "os"
    "path/filepath"
    "sort"
    "gorm.io/gorm"
)

// RunMigrations menjalankan semua file SQL di folder migrations secara berurutan
func RunMigrations(db *gorm.DB) error {
    // 1. Buat tabel migrations_history jika belum ada
    // 2. Baca semua file *.sql di folder migrations/
    // 3. Sort berdasarkan nama file (001_, 002_, dst)
    // 4. Skip file yang sudah ada di migrations_history
    // 5. Jalankan file SQL yang belum dieksekusi
    // 6. Catat ke migrations_history
}
```

### 3. Tabel `migrations_history`
```sql
CREATE TABLE IF NOT EXISTS migrations_history (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    filename     VARCHAR(255) UNIQUE NOT NULL,
    executed_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4. Panggil `RunMigrations` di `server/bootstrap.go`
```go
// Setelah koneksi database berhasil
if err := database.RunMigrations(db); err != nil {
    log.Fatal("Migration failed:", err)
}
```

### 5. Isi `001_init_schema.sql`
Salin semua DDL dari prompt 1.1 — semua 25 tabel lengkap dengan foreign key dan index.
