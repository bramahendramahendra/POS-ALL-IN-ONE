# Prompt 1.15 — API Contract Piutang

## Konteks
Piutang terjadi saat transaksi dilakukan dengan metode kredit (`is_credit = true`).
Pembayaran piutang bisa dilakukan parsial (dicicil).

## Endpoint

### GET /api/receivables
**Auth:** Bearer Token | Semua role  

**Query Params:**
```
?customer_id=1
?status=unpaid
?start_date=2024-01-01
?end_date=2024-01-31
?page=1&limit=20
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    {
      "id": 1,
      "transaction_code": "WEB-20240101-001",
      "customer_name": "Budi Santoso",
      "total_amount": 500000,
      "paid_amount": 200000,
      "remaining_amount": 300000,
      "status": "partial",
      "due_date": "2024-02-01"
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 20 }
}
```

### POST /api/receivables/:id/pay
**Deskripsi:** Catat pembayaran piutang  
**Auth:** Bearer Token | Semua role  

**Request Body:**
```json
{
  "payment_date": "2024-01-15",
  "amount": 200000,
  "payment_method": "cash",
  "notes": "Bayar sebagian"
}
```

**Validasi:**
- `amount` tidak boleh melebihi `remaining_amount`

**Proses:**
1. Simpan ke `receivable_payments`
2. Update `paid_amount` dan `remaining_amount` di `receivables`
3. Update status: jika `remaining = 0` → `paid`, jika > 0 → `partial`

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Pembayaran piutang berhasil dicatat",
  "data": {
    "paid_amount": 400000,
    "remaining_amount": 100000,
    "status": "partial"
  }
}
```
