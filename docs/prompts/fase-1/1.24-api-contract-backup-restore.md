# Prompt 1.24 — API Contract Backup & Restore

## Konteks
Backup dan restore database MySQL.
Di desktop, backup juga mencakup SQLite lokal.
File backup disimpan di server atau diunduh langsung.

## Endpoint

### POST /api/backup
**Deskripsi:** Buat backup database MySQL  
**Auth:** Bearer Token | Role: owner, admin  

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Backup berhasil dibuat",
  "data": {
    "filename": "backup_20240101_100000.sql",
    "size": "2.5 MB",
    "created_at": "2024-01-01T10:00:00Z"
  }
}
```

---

### GET /api/backup/download/:filename
**Deskripsi:** Download file backup  
**Auth:** Bearer Token | Role: owner, admin  

**Response:** File SQL sebagai download attachment

---

### GET /api/backup/list
**Deskripsi:** List file backup yang tersedia  
**Auth:** Bearer Token | Role: owner, admin  

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    { "filename": "backup_20240101_100000.sql", "size": "2.5 MB", "created_at": "2024-01-01T10:00:00Z" }
  ]
}
```

---

### POST /api/restore
**Deskripsi:** Restore database dari file backup  
**Auth:** Bearer Token | Role: admin  
**Content-Type:** multipart/form-data  

**Form Data:**
```
file: [file .sql]
```

**Response (200):**
```json
{ "code": "00", "status": true, "message": "Restore database berhasil" }
```

**Catatan:** Endpoint ini sangat berbahaya — pastikan hanya admin yang bisa akses dan ada konfirmasi di frontend.
