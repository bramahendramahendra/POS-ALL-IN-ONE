# Prompt 1.17 — API Contract Stock Mutations

## Konteks
Stock mutations mencatat setiap perubahan stok produk.
Tabel ini bersifat append-only (log) — tidak ada update/delete.
Mutasi terjadi otomatis saat: transaksi, pembelian, retur, void transaksi.

## Endpoint

### GET /api/stock-mutations
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?product_id=1
?mutation_type=in
?start_date=2024-01-01
?end_date=2024-01-31
?page=1&limit=20
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    {
      "id": 1,
      "product_id": 1,
      "product_name": "Kopi Susu",
      "mutation_type": "out",
      "quantity": 2,
      "stock_before": 52,
      "stock_after": 50,
      "reference_type": "transaction",
      "reference_id": 1,
      "notes": "Transaksi WEB-20240101-001",
      "user_name": "Kasir Satu",
      "created_at": "2024-01-01T10:00:00Z"
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 200 }
}
```

---

### GET /api/stock-mutations/product/:product_id
**Deskripsi:** Riwayat mutasi stok satu produk  
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?page=1&limit=20
```

**Response (200):** Sama dengan list, difilter by product_id

---

## Tipe Mutasi
| Tipe | Keterangan | Stok |
|---|---|---|
| `in` | Stok masuk (pembelian) | + |
| `out` | Stok keluar (transaksi) | - |
| `adjustment` | Penyesuaian manual | +/- |
| `void` | Void transaksi (rollback) | + |
| `return` | Retur supplier (approved) | - |

## Catatan
Endpoint ini hanya untuk membaca (GET).
Mutasi stok dibuat otomatis oleh backend saat operasi terkait terjadi.
Tidak ada endpoint POST/PUT/DELETE untuk stock mutations.
