# Prompt 1.2 — API Contract Auth & Session

## Konteks
Sistem menggunakan JWT dengan aturan **Single Active Session**:
- 1 akun hanya boleh login di 1 device dalam waktu yang sama
- Jika login di device baru, session lama otomatis diinvalidate
- Device lama akan mendapat response 401 pada request berikutnya

## Endpoint

### POST /api/auth/login
**Deskripsi:** Login user, generate JWT token  
**Auth:** Tidak perlu  

**Request Body:**
```json
{
  "username": "admin",
  "password": "admin123",
  "device_info": "desktop"
}
```

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Login berhasil",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_at": "2024-01-01T10:00:00Z",
    "user": {
      "id": 1,
      "username": "admin",
      "full_name": "Administrator",
      "role": "admin"
    }
  }
}
```

**Response Error (401):**
```json
{
  "code": "41",
  "status": false,
  "message": "Username atau password salah"
}
```

---

### POST /api/auth/logout
**Deskripsi:** Logout, hapus session aktif  
**Auth:** Bearer Token  

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Logout berhasil"
}
```

---

### POST /api/auth/refresh
**Deskripsi:** Refresh JWT token yang hampir expired  
**Auth:** Tidak perlu (gunakan refresh_token)  

**Request Body:**
```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIs..."
}
```

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Token berhasil diperbarui",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_at": "2024-01-01T10:00:00Z"
  }
}
```

---

### GET /api/auth/me
**Deskripsi:** Get data user yang sedang login  
**Auth:** Bearer Token  

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Success",
  "data": {
    "id": 1,
    "username": "admin",
    "full_name": "Administrator",
    "role": "admin",
    "is_active": true
  }
}
```

---

## Aturan Single Active Session
- Saat login, cek tabel `sessions` apakah user sudah punya session aktif
- Jika ada → hapus/invalidate session lama → buat session baru
- Setiap request dengan Bearer token → cek token di tabel `sessions`
- Jika token tidak ditemukan di `sessions` → return 401
- Device lama yang token-nya sudah dihapus → otomatis logout

## JWT Claims
```json
{
  "user_id": 1,
  "username": "admin",
  "role": "admin",
  "device_info": "desktop",
  "exp": 1234567890,
  "iat": 1234567890
}
```

## Response Error Umum
```json
{ "code": "41", "status": false, "message": "Token tidak valid atau sudah expired" }
{ "code": "41", "status": false, "message": "Sesi Anda telah berakhir karena login di perangkat lain" }
{ "code": "43", "status": false, "message": "Akun Anda tidak aktif" }
```
