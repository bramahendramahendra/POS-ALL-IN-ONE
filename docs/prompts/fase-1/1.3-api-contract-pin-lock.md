# Prompt 1.3 — API Contract PIN Lock

## Konteks
PIN digunakan oleh kasir sebagai keamanan tambahan saat meninggalkan sesi.
PIN disimpan sebagai bcrypt hash di kolom `pin_hash` tabel `users`.

## Endpoint

### GET /api/pin/check
**Deskripsi:** Cek apakah user sudah memiliki PIN  
**Auth:** Bearer Token  

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Success",
  "data": {
    "has_pin": true
  }
}
```

---

### POST /api/pin/set
**Deskripsi:** Set PIN baru (hanya jika belum punya PIN)  
**Auth:** Bearer Token  

**Request Body:**
```json
{
  "pin": "1234"
}
```

**Validasi:**
- PIN harus 4–6 digit angka
- Jika sudah punya PIN → return error, gunakan endpoint change

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "PIN berhasil disimpan"
}
```

---

### POST /api/pin/verify
**Deskripsi:** Verifikasi PIN (untuk unlock layar)  
**Auth:** Bearer Token  

**Request Body:**
```json
{
  "pin": "1234"
}
```

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "PIN benar"
}
```

**Response Error (401):**
```json
{
  "code": "41",
  "status": false,
  "message": "PIN salah"
}
```

---

### POST /api/pin/change
**Deskripsi:** Ubah PIN lama ke PIN baru  
**Auth:** Bearer Token  

**Request Body:**
```json
{
  "old_pin": "1234",
  "new_pin": "5678"
}
```

**Response Sukses (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "PIN berhasil diubah"
}
```

**Response Error (401):**
```json
{
  "code": "41",
  "status": false,
  "message": "PIN lama tidak sesuai"
}
```

---

## Aturan PIN
- PIN hanya berlaku untuk role `kasir`
- Format: 4–6 digit angka saja
- Disimpan sebagai bcrypt hash
- Tidak ada limit percobaan salah (untuk kemudahan kasir)
