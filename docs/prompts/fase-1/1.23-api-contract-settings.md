# Prompt 1.23 — API Contract Settings

## Konteks
Settings menyimpan konfigurasi toko yang bisa diubah lewat UI.
Data disimpan sebagai key-value di tabel `settings`.

## Key Settings yang Ada
| Key | Default | Keterangan |
|---|---|---|
| store_name | Toko Retail | Nama toko |
| store_address | (kosong) | Alamat toko |
| store_phone | (kosong) | No. telepon |
| store_email | (kosong) | Email toko |
| tax_enabled | 0 | Pajak aktif (0/1) |
| tax_percent | 11 | Persentase pajak |
| receipt_footer | Terima kasih | Footer struk |
| stock_notification_enabled | 1 | Notifikasi stok (0/1) |

## Endpoint

### GET /api/settings
**Deskripsi:** Get semua settings  
**Auth:** Bearer Token | Semua role  

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": {
    "store_name": "Toko Retail",
    "store_address": "Jl. Merdeka No. 1",
    "store_phone": "021-1234567",
    "store_email": "toko@email.com",
    "tax_enabled": "1",
    "tax_percent": "11",
    "receipt_footer": "Terima kasih telah berbelanja",
    "stock_notification_enabled": "1"
  }
}
```

---

### GET /api/settings/:key
**Deskripsi:** Get satu setting by key  
**Auth:** Bearer Token | Semua role  

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": { "key": "store_name", "value": "Toko Retail" }
}
```

---

### POST /api/settings
**Deskripsi:** Simpan satu atau banyak settings sekaligus (upsert)  
**Auth:** Bearer Token | Role: owner, admin  

**Request Body:**
```json
{
  "store_name": "Toko Makmur",
  "store_address": "Jl. Baru No. 5",
  "tax_enabled": "1",
  "tax_percent": "11"
}
```

**Response (200):**
```json
{ "code": "00", "status": true, "message": "Settings berhasil disimpan" }
```

---

### POST /api/settings/reset
**Deskripsi:** Reset semua settings ke nilai default  
**Auth:** Bearer Token | Role: admin  

**Response (200):**
```json
{ "code": "00", "status": true, "message": "Settings berhasil direset ke default" }
```
