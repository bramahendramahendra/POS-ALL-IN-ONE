# Prompt 1.25 — API Contract Sync Center

## Konteks
Sync Center menangani sinkronisasi data antara Desktop offline dan server online.
Ada dua fungsi utama:
1. **Konflik** — data yang diedit di desktop offline dan online secara bersamaan
2. **Antrian Sync** — data desktop offline yang menunggu disinkronkan ke server

## Endpoint

### GET /api/sync/conflicts
**Deskripsi:** List semua konflik yang belum diselesaikan  
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?entity_type=product
?status=pending
?page=1&limit=20
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    {
      "id": 1,
      "entity_type": "product",
      "entity_id": 5,
      "desktop_data": { "name": "Kopi Susu", "selling_price": 15000 },
      "online_data": { "name": "Kopi Susu Gula", "selling_price": 18000 },
      "desktop_time": "2024-01-01T08:30:00Z",
      "online_time": "2024-01-01T09:15:00Z",
      "status": "pending"
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 3 }
}
```

---

### POST /api/sync/conflicts/:id/resolve
**Deskripsi:** Selesaikan konflik dengan memilih versi yang benar  
**Auth:** Bearer Token | Role: owner, admin  

**Request Body:**
```json
{
  "resolution": "desktop"
}
```
`resolution` bisa `desktop` atau `online`

**Proses:**
- Jika `desktop` → terapkan `desktop_data` ke MySQL
- Jika `online` → abaikan `desktop_data`, data online tetap berlaku
- Update status konflik menjadi `resolved`
- Catat siapa yang resolve dan kapan

**Response (200):**
```json
{ "code": "00", "status": true, "message": "Konflik berhasil diselesaikan" }
```

---

### GET /api/sync/queue
**Deskripsi:** List antrian sync dari semua device desktop  
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?device_id=desktop-001
?status=pending
?page=1&limit=20
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    {
      "id": 1,
      "device_id": "desktop-001",
      "entity_type": "product",
      "entity_id": 5,
      "action": "update",
      "status": "pending",
      "retry_count": 0,
      "created_at": "2024-01-01T08:30:00Z"
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 10 }
}
```

---

### POST /api/sync/push
**Deskripsi:** Desktop mengirim antrian sync ke server  
**Auth:** Bearer Token | Device Desktop  

**Request Body:**
```json
{
  "device_id": "desktop-001",
  "items": [
    {
      "entity_type": "transaction",
      "entity_id": null,
      "action": "create",
      "payload": { "transaction_code": "DSK-20240101-001", "..." : "..." }
    }
  ]
}
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Sync diterima",
  "data": {
    "processed": 3,
    "conflicts": 1,
    "failed": 0
  }
}
```

---

### GET /api/sync/history
**Deskripsi:** Riwayat semua aktivitas sync per device  
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?device_id=desktop-001
?start_date=2024-01-01
?page=1&limit=20
```

**Response (200):**
```json
{
  "code": "00", "status": true, "message": "Success",
  "data": [
    {
      "id": 1,
      "device_id": "desktop-001",
      "entity_type": "product",
      "action": "update",
      "status": "synced",
      "synced_at": "2024-01-01T09:00:00Z"
    }
  ]
}
```
