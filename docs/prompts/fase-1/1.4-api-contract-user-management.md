# Prompt 1.4 — API Contract User Management

## Konteks
Manajemen user hanya bisa dilakukan oleh role `owner` dan `admin`.
Role yang tersedia: `owner`, `admin`, `kasir`.

## Endpoint

### GET /api/users
**Deskripsi:** List semua user  
**Auth:** Bearer Token | Role: owner, admin  

**Query Params:**
```
?search=nama_atau_username
?role=kasir
?is_active=1
```

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Success",
  "data": [
    {
      "id": 1,
      "username": "admin",
      "full_name": "Administrator",
      "role": "admin",
      "is_active": true,
      "created_at": "2024-01-01T00:00:00Z"
    }
  ]
}
```

---

### GET /api/users/:id
**Deskripsi:** Detail satu user  
**Auth:** Bearer Token | Role: owner, admin  

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Success",
  "data": {
    "id": 1,
    "username": "admin",
    "full_name": "Administrator",
    "role": "admin",
    "is_active": true,
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

---

### POST /api/users
**Deskripsi:** Tambah user baru  
**Auth:** Bearer Token | Role: owner, admin  

**Request Body:**
```json
{
  "username": "kasir1",
  "password": "password123",
  "full_name": "Kasir Satu",
  "role": "kasir"
}
```

**Validasi:**
- `username`: required, unik, min 3 karakter, alphanumeric
- `password`: required, min 6 karakter
- `full_name`: required
- `role`: required, salah satu dari owner/admin/kasir

**Response (201):**
```json
{
  "code": "21",
  "status": true,
  "message": "User berhasil ditambahkan",
  "data": { "id": 2 }
}
```

---

### PUT /api/users/:id
**Deskripsi:** Update data user  
**Auth:** Bearer Token | Role: owner, admin  

**Request Body:**
```json
{
  "full_name": "Kasir Satu Updated",
  "role": "kasir",
  "password": "newpassword123"
}
```

**Catatan:** `password` opsional — kosongkan jika tidak ingin ganti password

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "User berhasil diupdate"
}
```

---

### DELETE /api/users/:id
**Deskripsi:** Hapus user  
**Auth:** Bearer Token | Role: owner, admin  

**Validasi:**
- Tidak bisa hapus diri sendiri
- Tidak bisa hapus user yang sedang login di device lain

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "User berhasil dihapus"
}
```

---

### PATCH /api/users/:id/toggle-status
**Deskripsi:** Aktifkan atau nonaktifkan user  
**Auth:** Bearer Token | Role: owner, admin  

**Response (200):**
```json
{
  "code": "00",
  "status": true,
  "message": "Status user berhasil diubah",
  "data": { "is_active": false }
}
```
