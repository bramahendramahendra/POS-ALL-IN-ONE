# Prompt 0.2 — Bersihkan Backend dari Integrasi yang Tidak Dipakai

## Konteks
Backend Go GIN yang ada saat ini berasal dari project perbankan (BRI).
Terdapat banyak integrasi eksternal yang tidak diperlukan untuk aplikasi POS:
- ESB (Enterprise Service Bus) — integrasi perbankan
- BRIGate — payment gateway perbankan
- ESB Monolith — layanan monolith perbankan
- Bristars — integrasi khusus BRI
- Minio/S3 — object storage (tidak dipakai POS)
- Redis — optional, belum tentu dipakai POS

## Struktur Backend Saat Ini
```
backend/
├── config/
│   ├── config.go
│   ├── config_dev.json       ← Berisi config ESB, BRIGate, Minio, Redis
│   └── config_local.json
├── domain/
│   ├── auth/
│   └── sample/
├── pkg/
│   ├── database/
│   ├── logger/
│   ├── jwt/
│   ├── bcrypt/
│   ├── encryptor/
│   ├── binder/
│   ├── redis/                ← Mungkin tidak dipakai
│   ├── minio/                ← Tidak dipakai
│   ├── janitor/
│   └── transport/            ← Untuk ESB/BRIGate, tidak dipakai
├── dto/
│   ├── response_dto.go
│   ├── error_dto.go
│   ├── filter_request_dto.go
│   ├── esb.go                ← Hapus
│   └── brigate.go            ← Hapus
└── server/
    └── bootstrap.go          ← Berisi init Minio, Redis, ESB transport
```

## Tugas

### 1. Hapus file DTO yang tidak relevan
- Hapus `dto/esb.go`
- Hapus `dto/brigate.go`

### 2. Hapus atau kosongkan package yang tidak dipakai
- Hapus `pkg/minio/` (tidak dipakai POS)
- Hapus `pkg/transport/` (untuk ESB/BRIGate)
- Pertahankan `pkg/redis/` tapi jangan wajibkan (jadikan optional)

### 3. Bersihkan `server/bootstrap.go`
Hapus inisialisasi yang tidak diperlukan:
- Hapus init Minio
- Hapus init ESB transport client
- Hapus init BRIGate transport client
- Pertahankan: Database, Logger, JWT, Redis (optional)

### 4. Bersihkan `routes/public_routes.go` dan `routes/protected_routes.go`
- Hapus sample routes yang ada (user integration, ESB routes)
- Kosongkan dan siapkan untuk routes POS

### 5. Hapus `domain/sample/` (sudah tidak diperlukan setelah dipahami polanya)
- Atau rename menjadi `domain/_sample_reference/` sebagai referensi saja

### 6. Update go.mod
Jalankan `go mod tidy` untuk membersihkan dependency yang tidak terpakai setelah penghapusan.

## Output yang Diharapkan
Backend yang bersih, hanya berisi komponen yang relevan untuk POS.
Tidak ada error compile setelah pembersihan.
Jalankan `go build ./...` untuk verifikasi.
