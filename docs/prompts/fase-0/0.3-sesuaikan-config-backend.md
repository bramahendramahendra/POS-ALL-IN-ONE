# Prompt 0.3 — Sesuaikan Config Backend untuk POS

## Konteks
Setelah pembersihan di prompt 0.2, config backend perlu disesuaikan
dari config perbankan menjadi config yang relevan untuk aplikasi POS.

## Config Saat Ini (config_dev.json) — Perlu Diganti
```json
{
  "SecretKey": "My secret key",
  "TokenExpire": 7200,
  "Timezone": "Asia/Jakarta",
  "Database": { ... },
  "RedisLocal": { ... },
  "Minio": { ... },          ← Hapus
  "ESBConf": { ... },        ← Hapus
  "BRIgateConf": { ... },    ← Hapus
  "EsbMonolithConf": { ... } ← Hapus
}
```

## Tugas

### 1. Update `config/config_dev.json`
Buat config baru yang sesuai untuk POS:
```json
{
  "AppName": "POS Retail API",
  "AppVersion": "1.0.0",
  "AppPort": "8080",
  "Timezone": "Asia/Jakarta",
  "ReleaseMode": "dev",

  "SecretKey": "pos-retail-secret-key-dev",
  "TokenExpire": 28800,
  "RefreshTokenExpire": 604800,

  "Database": {
    "Type": "mysql",
    "Host": "127.0.0.1",
    "Port": "3306",
    "User": "root",
    "Password": "",
    "Database": "pos_retail_db",
    "MaxOpenConns": 50,
    "MaxIdleConns": 10,
    "MaxLifetime": 300
  },

  "CorsAllowOrigins": [
    "http://localhost:3000",
    "http://localhost:5173",
    "http://localhost:8080"
  ],

  "LogPath": "logs",
  "MaxLogAge": 30
}
```

### 2. Update `config/config.go`
Sesuaikan struct Config dengan field yang baru:
```go
type Config struct {
    AppName              string         `json:"AppName"`
    AppVersion           string         `json:"AppVersion"`
    AppPort              string         `json:"AppPort"`
    Timezone             string         `json:"Timezone"`
    ReleaseMode          string         `json:"ReleaseMode"`
    SecretKey            string         `json:"SecretKey"`
    TokenExpire          int            `json:"TokenExpire"`
    RefreshTokenExpire   int            `json:"RefreshTokenExpire"`
    Database             DatabaseConfig `json:"Database"`
    CorsAllowOrigins     []string       `json:"CorsAllowOrigins"`
    LogPath              string         `json:"LogPath"`
    MaxLogAge            int            `json:"MaxLogAge"`
}

type DatabaseConfig struct {
    Type         string `json:"Type"`
    Host         string `json:"Host"`
    Port         string `json:"Port"`
    User         string `json:"User"`
    Password     string `json:"Password"`
    Database     string `json:"Database"`
    MaxOpenConns int    `json:"MaxOpenConns"`
    MaxIdleConns int    `json:"MaxIdleConns"`
    MaxLifetime  int    `json:"MaxLifetime"`
}
```

### 3. Update `middleware/cors_middleware.go`
Gunakan `CorsAllowOrigins` dari config (tidak hardcode):
```go
func Cors(allowOrigins []string) gin.HandlerFunc {
    return func(c *gin.Context) {
        origin := c.Request.Header.Get("Origin")
        for _, allowed := range allowOrigins {
            if origin == allowed {
                c.Header("Access-Control-Allow-Origin", origin)
                break
            }
        }
        // ... rest of CORS logic
    }
}
```

### 4. Buat `config/config_prod.json`
Sama dengan config_dev tapi:
- `ReleaseMode`: "release"
- `Database.Host`: sesuaikan dengan VPS
- `SecretKey`: ganti dengan key yang kuat
- `CorsAllowOrigins`: hanya domain production

### 5. Update `.env`
```env
GIN_MODE=debug
RELEASE_MODE=dev
APP_PORT=8080
```

## Output yang Diharapkan
- Config bersih tanpa referensi perbankan
- Struct config sesuai kebutuhan POS
- `go build ./...` berhasil tanpa error
