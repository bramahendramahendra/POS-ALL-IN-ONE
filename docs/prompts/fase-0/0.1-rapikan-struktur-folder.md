# Prompt 0.1 — Rapikan Struktur Folder Project POS

## Konteks
Saya memiliki project POS Multi-Platform di folder `Project_POS/`.
Saat ini kondisi folder sebagai berikut:
- Folder desktop Electron ada di `destop/` (typo, seharusnya `desktop/`)
- Folder `backend/` sudah ada berisi Go GIN
- Folder `web/` sudah ada tapi kosong
- Folder `android/` sudah ada tapi kosong
- Folder `docs/` berisi dokumentasi

## Tugas
Lakukan hal berikut:

1. **Rename folder** `destop/` menjadi `desktop/`

2. **Pastikan struktur folder final** seperti ini:
```
Project_POS/
├── desktop/        ← Electron app (dari destop/)
├── backend/        ← Go GIN backend
├── web/            ← Web app (akan diisi fase 4)
├── android/        ← Android Capacitor (akan diisi fase 5)
├── shared/         ← Buat folder ini (shared assets/utils antar platform)
└── docs/
    ├── prompts/    ← Folder prompt per fase
    ├── Rancangan Sistem POS Multi-Platform.md
    └── List Fase Pengerjaan POS Multi-Platform.md
```

3. **Buat folder** `shared/` jika belum ada

4. **Buat file** `shared/README.md` dengan isi:
```
# Shared
Folder ini berisi asset dan utility yang digunakan bersama oleh semua platform.
```

5. **Update `.gitignore`** di root project, pastikan mencakup:
```
# Dependencies
node_modules/
desktop/node_modules/
web/node_modules/
android/node_modules/

# Build output
desktop/dist/
web/dist/
android/dist/
android/android/
*.apk

# Environment
.env
*.env.local
backend/config/config_local.json
backend/config/config_prod.json

# Logs
backend/logs/
desktop/logs/
*.log

# Database
desktop/pos-retail.db
desktop/backups/

# OS
.DS_Store
Thumbs.db
```

## Output yang Diharapkan
Struktur folder yang rapi dan siap untuk pengerjaan multi-platform.
