# Prompt 6.2 — Simpan Konflik ke Sync Center

## File Diubah
`backend/domain/sync/service/sync_service.go`, `backend/domain/sync/repo/sync_repo.go`

## Implementasi SaveConflict

```go
// sync_service.go
func (s *syncService) SaveConflict(item SyncItem) (int64, error) {
    // Ambil data online saat ini sebagai snapshot
    onlineData, _ := s.repo.GetRawByEntityAndID(item.Entity, item.ServerID)

    conflict := model.SyncConflict{
        Entity:      item.Entity,
        EntityID:    item.ServerID,
        LocalID:     item.LocalID,
        DeviceID:    item.DeviceID,
        DesktopData: item.Payload,   // JSON data dari desktop
        OnlineData:  onlineData,     // JSON data dari MySQL
        Status:      "pending",
    }

    return s.repo.InsertConflict(conflict)
}
```

## Endpoint List Konflik (untuk Sync Center)

```go
// GET /sync/conflicts?status=pending&page=1&limit=20
func (h *SyncHandler) GetConflicts(c *gin.Context) {
    status := c.DefaultQuery("status", "pending")
    page, _  := strconv.Atoi(c.DefaultQuery("page", "1"))
    limit, _ := strconv.Atoi(c.DefaultQuery("limit", "20"))

    conflicts, total, _ := h.service.GetConflicts(status, page, limit)
    c.JSON(200, gin.H{
        "data":  conflicts,
        "total": total,
        "page":  page,
        "limit": limit,
    })
}
```
