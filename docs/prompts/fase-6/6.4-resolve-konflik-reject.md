# Prompt 6.4 — Resolve Konflik: Reject

## Konsep
**Reject** = data versi **online** tetap dipakai → data dari desktop diabaikan. Jika entity adalah transaksi, stok yang sempat dikurangi oleh transaksi offline dikembalikan.

## Implementasi

```go
// sync_service.go
func (s *syncService) rejectDesktopVersion(conflict *model.SyncConflict, resolvedBy int64) error {
    // Data online TIDAK diubah — tetap seperti adanya di MySQL

    // Jika entity adalah transaksi, kembalikan stok
    if conflict.Entity == "transaction" {
        var desktopData dto.TransactionData
        json.Unmarshal([]byte(conflict.DesktopData), &desktopData)

        if err := s.returnStockFromTransaction(desktopData); err != nil {
            return err
        }
    }

    // Tandai konflik sebagai resolved dengan action reject
    return s.repo.MarkResolved(conflict.ID, "reject", resolvedBy)
}

func (s *syncService) returnStockFromTransaction(tx dto.TransactionData) error {
    // Kembalikan stok setiap item dalam transaksi yang ditolak
    for _, item := range tx.Items {
        if err := s.productRepo.IncrementStock(item.ProductID, item.Quantity); err != nil {
            return err
        }
        // Catat mutasi stok: REJECT_SYNC
        s.stockMutationRepo.Insert(model.StockMutation{
            ProductID:   item.ProductID,
            Type:        "REJECT_SYNC",
            Quantity:    item.Quantity,
            Note:        fmt.Sprintf("Reject sync konflik transaksi offline %s", tx.LocalID),
        })
    }
    return nil
}
```

## Catatan Penting
- Reject tidak menghapus data dari MySQL — data online tetap ada
- Stok dikembalikan via `IncrementStock` (bukan insert transaksi baru)
- Setiap pengembalian stok dicatat di `stock_mutations` dengan type `REJECT_SYNC` untuk audit trail
- Jika entity bukan transaksi (misal: pengeluaran), reject hanya mengabaikan data desktop tanpa efek stok

## Output yang Diharapkan
Reject konflik mengabaikan data desktop, data online tetap, stok dikembalikan jika transaksi ditolak, semua tercatat di stock_mutations.
