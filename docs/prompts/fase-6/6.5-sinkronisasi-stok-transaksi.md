# Prompt 6.5 — Sinkronisasi Stok dari Transaksi

## Konsep
Saat desktop mengirim transaksi offline ke backend dan **tidak ada konflik**, transaksi langsung di-apply. Stok harus dikurangi di MySQL secara atomik untuk menghindari race condition dengan transaksi lain yang masuk bersamaan.

## Implementasi di sync_service.go

```go
func (s *syncService) ApplyTransaction(tx dto.TransactionData) (int64, error) {
    // Gunakan DB transaction untuk atomisitas
    dbTx, err := s.db.Begin()
    if err != nil {
        return 0, err
    }
    defer dbTx.Rollback()

    // 1. Cek stok mencukupi untuk semua item (SELECT FOR UPDATE)
    for _, item := range tx.Items {
        var currentStock int
        err := dbTx.QueryRow(
            `SELECT stock FROM products WHERE id = ? FOR UPDATE`,
            item.ProductID,
        ).Scan(&currentStock)
        if err != nil {
            return 0, err
        }
        if currentStock < item.Quantity {
            return 0, fmt.Errorf("stok produk %d tidak mencukupi (%d tersedia, butuh %d)",
                item.ProductID, currentStock, item.Quantity)
        }
    }

    // 2. Insert transaksi
    transactionID, err := s.insertTransaction(dbTx, tx)
    if err != nil {
        return 0, err
    }

    // 3. Kurangi stok setiap item + catat mutasi
    for _, item := range tx.Items {
        _, err = dbTx.Exec(
            `UPDATE products SET stock = stock - ? WHERE id = ?`,
            item.Quantity, item.ProductID,
        )
        if err != nil {
            return 0, err
        }

        // Catat mutasi stok
        _, err = dbTx.Exec(`
            INSERT INTO stock_mutations (product_id, type, quantity, reference_id, reference_type, note, created_at)
            VALUES (?, 'SALE', ?, ?, 'transaction', ?, NOW())
        `, item.ProductID, -item.Quantity, transactionID, fmt.Sprintf("Sync offline tx %s", tx.LocalID))
        if err != nil {
            return 0, err
        }
    }

    // 4. Commit
    if err = dbTx.Commit(); err != nil {
        return 0, err
    }

    return transactionID, nil
}
```

## Catatan
- `SELECT FOR UPDATE` memastikan tidak ada transaksi lain yang mengubah stok di antara cek dan update
- Jika stok tidak mencukupi (bisa terjadi jika ada transaksi online lebih dulu), item dikembalikan sebagai konflik ke desktop
- Seluruh proses dalam 1 DB transaction — gagal di satu item berarti seluruh transaksi di-rollback
