# Prompt 6.7 — Notifikasi Konflik ke Owner/Admin

## Konsep
Saat ada konflik baru masuk dari device desktop, Owner/Admin yang sedang buka Sync Center di web/desktop mendapat notifikasi tanpa perlu refresh halaman.

## Metode: Polling (tanpa WebSocket)
Gunakan **polling ringan** setiap 30 detik — lebih sederhana dari WebSocket, cukup untuk kebutuhan POS.

## Backend: Endpoint Count Konflik Pending

```go
// GET /sync/conflicts/count — kembalikan jumlah konflik pending
func (h *SyncHandler) GetConflictCount(c *gin.Context) {
    count, _ := h.service.CountPendingConflicts()
    c.JSON(200, gin.H{"count": count})
}
```

## Frontend: notification-poller.js

```javascript
// notification-poller.js
class ConflictNotificationPoller {
    constructor() {
        this.lastCount = 0;
        this.interval = null;
    }

    start() {
        // Hanya aktif untuk role owner dan admin
        const user = getCurrentUser();
        if (!user || !['owner', 'admin'].includes(user.role)) return;

        this.interval = setInterval(() => this._check(), 30000);
        this._check(); // cek awal
    }

    async _check() {
        if (!connectionMonitor.isOnline) return;
        try {
            const { count } = await apiClient.get('/sync/conflicts/count');
            this._updateBadge(count);

            // Notifikasi jika jumlah bertambah
            if (count > this.lastCount && this.lastCount > 0) {
                this._showNotification(count - this.lastCount);
            }
            this.lastCount = count;
        } catch {}
    }

    _updateBadge(count) {
        const badge = document.getElementById('sync-center-badge');
        if (!badge) return;
        badge.textContent = count > 0 ? count : '';
        badge.style.display = count > 0 ? 'inline' : 'none';
    }

    _showNotification(newCount) {
        showToast(`${newCount} konflik sync baru menunggu persetujuan`, 'warning', {
            action: { label: 'Lihat', onClick: () => navigate('/sync-center') }
        });
    }

    stop() {
        clearInterval(this.interval);
    }
}

const conflictPoller = new ConflictNotificationPoller();
conflictPoller.start();
```

## Catatan
- Polling 30 detik cukup untuk kebutuhan POS — tidak perlu realtime WebSocket
- Badge muncul di menu sidebar Sync Center jika ada konflik pending
- Toast notification hanya muncul jika jumlah konflik **bertambah** (bukan saat pertama load)
- Di masa depan bisa diganti ke Server-Sent Events (SSE) jika dibutuhkan realtime

## Output yang Diharapkan
Owner/Admin mendapat notifikasi badge dan toast saat ada konflik sync baru, tanpa perlu refresh halaman.
