# Prompt 6.3 — Resolve Konflik: Approve

## Endpoint
`POST /sync/conflicts/:id/resolve` dengan `body: { "action": "approve" }`

## Konsep
**Approve** = data versi **desktop** yang dipakai → data online di MySQL di-overwrite dengan data desktop.

## Implementasi

```go
// sync_service.go
func (s *syncService) ResolveConflict(conflictID int64, action string, resolvedBy int64) error {
    conflict, err := s.repo.GetConflictByID(conflictID)
    if err != nil || conflict == nil {
        return errors.New("konflik tidak ditemukan")
    }
    if conflict.Status == "resolved" {
        return errors.New("konflik sudah diselesaikan")
    }

    if action == "approve" {
        return s.applyDesktopVersion(conflict, resolvedBy)
    } else if action == "reject" {
        return s.rejectDesktopVersion(conflict, resolvedBy) // lihat 6.4
    }
    return errors.New("action tidak valid: gunakan 'approve' atau 'reject'")
}

func (s *syncService) applyDesktopVersion(conflict *model.SyncConflict, resolvedBy int64) error {
    // Parse desktop_data JSON
    var desktopData map[string]interface{}
    json.Unmarshal([]byte(conflict.DesktopData), &desktopData)

    // Apply ke tabel yang sesuai berdasarkan entity
    switch conflict.Entity {
    case "transaction":
        if err := s.transactionRepo.UpdateFromSync(conflict.EntityID, desktopData); err != nil {
            return err
        }
    case "expense":
        if err := s.expenseRepo.UpdateFromSync(conflict.EntityID, desktopData); err != nil {
            return err
        }
    // tambah entity lain sesuai kebutuhan
    }

    // Tandai konflik sebagai resolved
    return s.repo.MarkResolved(conflict.ID, "approve", resolvedBy)
}
```

## Repo — MarkResolved

```go
// sync_repo.go
func (r *syncRepo) MarkResolved(id int64, action string, resolvedBy int64) error {
    query := `
        UPDATE sync_conflicts
        SET status = 'resolved', resolved_action = ?, resolved_by = ?, resolved_at = NOW()
        WHERE id = ?
    `
    _, err := r.db.Exec(query, action, resolvedBy, id)
    return err
}
```

## Handler

```go
// POST /sync/conflicts/:id/resolve
func (h *SyncHandler) ResolveConflict(c *gin.Context) {
    id, _ := strconv.ParseInt(c.Param("id"), 10, 64)
    var req struct {
        Action string `json:"action" binding:"required,oneof=approve reject"`
    }
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    userID := c.GetInt64("user_id") // dari middleware auth
    if err := h.service.ResolveConflict(id, req.Action, userID); err != nil {
        c.JSON(422, gin.H{"error": err.Error()})
        return
    }

    c.JSON(200, gin.H{"message": "Konflik berhasil diselesaikan"})
}
```

## Output yang Diharapkan
Approve konflik menimpa data online dengan versi desktop, konflik ditandai resolved dengan audit trail (who + when).
